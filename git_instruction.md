# Работа с Git
![logo.png](logo.png)
## 1. Проверка наличия установленного Git
В терминале выполнить команду `git version`.

Если Git установлен, появится сообщение с информацией о версии программы. Иначе будет сообщение об ошибке.

## 2. Установка Git
Загружаем последнюю версию Git с сайта 
Устанавливаем с настройками по умолчанию.

## 3. Настройка Git
При первом использовании Git необходимо представиться. Для этого необходимо в терминале выполнить две команды:
```
git config --global user.name «Ваше имя английскими буквами»
git config --global user.email ваша почта@example.com
```
## 4. Инициализация репозитория.
Получить репозиторий можно получить двумя способами:

1. В терминале переходим к папке, в которой хотим создать репозиторий. Выполняем команду:
```
git init
```
2. Клонировать существующий репозиторий Git из любого места. Сделать это можно так:
```
git clone <адрес репозитория> 
```

## 5. Запись изменений в репозиторий
После внесения изменений в наш репозиторий необходимо записать изменения, воспользовавшись комбинацией команд.
Сначала добавить содержимое каталога с помощью команды:
```
git add <название файла> (можно начать вводить имя файла, после чего нажать на Tab и выбрать нужный нам)
```
После зафиксировать изменения с помощью команды:
```
git commit -m "наш комментарий"
```

## 6. Просмотр истории коммитов
Чтобы увидеть все то, что увидеть всю историю с нашими сохранениями, необходимо ввести команду:
```
git log (для отображения в виде графика вводим git log --graph)
```

## 7. Перемещение между сохранениями
Для того, чтобы переключаться между нужными версиями коммитов в репозитории, нужно задать команду в терминале:
```
git checkout <идентификатор изменения>
```
А для того, чтобы вернуться в то место, где ведется работа воспользоваться командой:
```
git checkout
```
## 8. Игнорирование файлов
 Для того, чтобы исключить из отслеживания репозиторий файлы и папки, необходимо создать там файл с названием ****.gitignore*** и записать в него название файлов или шаблоны, соответсвующие таким файлам и папкам.

## 9. Создание веток в Git.
Ветка в Git - это простой перемещаемый указатель на один из коммитов, обычно последний в цепочке коммитов. По умолчанию имя основной ветки Git - master

Создать ветку можно командой:
```
git branch <название новой ветки>
```
В результате создается новый указатель на текущий коммит.
Есть также и другие команды, воспроизводящие данную команду, а именно:
```
git checkout -b <название новой ветки>
git switch -b <название новой ветки>
```
## 10. Слияние веток и разрешение конфликтов
Для того, чтобы перенести изменения из одной ветки в другую, необходимо для начала переключаться на основную ветку с помощью команды `git checkout <название нужной нам ветки>`.

После убедиться, что мы находимся именно в ней с помощью `git branch`. Зеленым светом будет подсвечиваться та ветка, в которой мы находимся.

Далее присоединения нужной ветки к текущей в адрес терминала вводим команду:
```
git merge <название той ветки, которую хотим присоединить>
```
Бывают случаи, когда мы внесли изменения в двух ветках сразу. И программа не понимает, что важнее для нас. В этом случае она предлагает нам несколько решений по устранению конфликтов. Необходимо выбрать интересующее нас.

После необходимо необходимо сделать коммит слияния с помощью команды: 
```
git commit -am "Наш комментарий"
```
При необходимости можно отменить слияние веток с помощью следующих команд следующих команд:
```
git reset --merge
git reset --abort
```
## 11. Удаление веток
Для удаления ненужных веток необходимо ввести команду:
```
git branch -d <название удаляемой ветки>
```
После чего в терминале появится сообщение о том, что ветка удалена, а после ввода команды `git branch` ветка перестанет отображаться.

В случае если мы не слили изменения с нашей текущей веткой, но пытаемся удалить дополнительную ветку, то появится ошибка с информацией о том, что данная ветка не слита с основной и предлагает возможные варианты:
* слиять с основной веткой,
* удалить без слияния с помощью команды принудительного удаления `git branch -D <название удаляемой ветки>`.

## 12. Создание удаленного репозитория
Для создания удаленного репозитория необходимо перейти на сайт __GitHub__, после чего пройти регистрацию или залогиниться, если репозиторий уже есть.
Далее можно добавить репозиторий с помощью нажатия на кнопку + (вверху страницы) и выбрать подходящий нам вид репозитория.
После создания репозитория мы попадаем на страницу с ссылкой репозитория и инструкцией по добавлению репозитория в локальный.

## 13. Связка локального репозитория с удаленным
После создания удаленного и локального репозитория, необходимо вернуться к локальному и последовательно ввести следующие команды для связки репозиториев:
```
echo "# -" >> <Название удаленного репозитория>
git init
git add <Название удаленного репозитория>
git commit -m "first commit"
git branch -M main
git remote add <имя для репозитория> <url-адрес для репозитория>
git push -u origin main
```
После выполненных команд переходим обратно на страницу GitHub, обновляем и видим наш репозиторий.

## 14. Перенос изменений из удаленного репозитория в локальный.
Чтобы получить внесенные изменения из удаленного репозитория в локальный необходимо перейти в программу **Visual Studio Code** и задать команду:
```
git pull
```
## 15. Перенос изменений из локального репозитория в удаленный.
Чтобы отправить данные в удаленный репозиторий, необходимо ввести команду `git push`. При первой отправке изменений необходимо будет пройти авторизацию.

## 16. Работа с чужими репозиториями.
Для нахождения интересующего проекта на GitHub можно воспользоваться поисковой строкой, введя название или воспользоваться уже существующей ссылкой проекта.

Необходимо создать копию репозитория, воспользовавшись **Fork**, следом перейти к файлу и нажать на кнопку **<> Code** скопировать ссылку проекта и перейти на локальный репозиторий и получить данные удаленного репозитория, воспользовавшись командой `Git clone <адрес ссылки>`.
Чтобы сохранить файл под другим именем можно вести команду `Git clone <адрес ссылки>` <новое название>.

Далее в загруженном файле необходимо создать новую ветку, воспользовавшись командой `git branch <название новой ветки>`, после перейдя на нашу ветку. Следом можно вносить изменения, сохранить, создать ветку в удаленном репозитории с помощью команды `git push --set-upstream origin <название нашей ветки>` и отправить данные в удаленный репозиторий.

Далее в удаленном репозитории переходим на созданную нами ветку в проекте, предложить внесенные изменения мы можем с помощью кнопки **pull requests**. Если кнопка не отобразилась, то в таком случае мы переходим во вкладу *pull requests*, нажимаем на кнопку **new pull request**, выбираем нашу ветку, выбираем кнопку **create pull request**, в открывшейся вкладке вводим сообщение автору проекта и нажимаем на кнопку **create pull request**. 

Далее автор может проверить предложенные изменения на вкладке *pull requests*, проверить новые данные в локальном репозитории и выполнить слияние.
