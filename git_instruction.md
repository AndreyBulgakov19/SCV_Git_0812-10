# Работа с git #
## 1. Проверка наличия установленного git
В терминале выполнить команду `git version`.

Если git установлен, появится сообщение с информацией о версии программы. Иначе будет сообщение об ошибке

## 2. Установка git
Загружаем последнюю версию git с сайта *https:/git-scm.com/downloads*
Устанавливаем с настройками по умолчанию

## 3. Настройка git
При первом использовании git нужно представиться. 
Для этого в терминале нужно выполнить две команды:
```
git config --global user.name «Ваше имя английскими буквами»

git config -- global user. email вашапочта@example.com
```
## 4. Инициализация репозитория
1. Создаем папку для работы и инициализируем ее в git, используя команду 

`git init`

таким образом мы создали репозиторий для сохранения всех изменений.

2. Клонировать существующий репозиторий из любого места. Сделать это можно так:

`git clone <адрес репозитория>`

## 5. Добавление файла отслеживания
Для добавления файла в отслеживание выполняем команду 

`git add имя файла`

Проверяем статус файла используя команду

*git status*

## 6. Сохранение изменений
Для сохранения внесенных изменений файла в git репозиторий, сначала добавляем измененный файл

`git add имя файла`

затем сохраняем изменения 

`git commit -m "Комментарий изменений"`

```
Если нужно внести изменения во все файлы без исключений, можно использовать комманду:

git commit -a -m "комментарий изменений"
```

## 7. История сохранений
Для вывода на экран истории всех сохранений с их хеш-кодами в хронологическом порядке вводим команду

***git log***

## 8. Перемещение между сохранениями
Как видно из заголовка, между сохранениями можно перемещаться. Делаем это, используя команду 

`git checkout` и добавляем из Имени commit-а минимум 4 первых символа

Пример `git checkout 3e13e`

```
после перемещения по commit-ам нужно вернуться в актуальное состояние по команде 

git checkout master 
или 
git switch -
```
## 9. Игнорирование файлов в папке отслеживаемой git

Для того, чтобы исключить из отслеживания в репозитории файлы и папки,необходимо создать там файл ***.gitignore*** и записать в него их название или шаблоны, соответствующие названиям файлов и/или папкам.

## 10. Создание веток в git
Ветка в git это простой перемещаемый указатель на один из коммитов, обычно последний в цепочке коммитов.
По умолчанию имя основной ветки в git - master.

Cоздать ветку можно командой
```
git branch имя_новой_ветки
```
В результате создается новый указатель на текущий коммит.

*** Важно, что новые ветки к главной ветке добавляют, находясь в самой (master/main) ветке ***

## 11. Перемещение на ветку

Перемещение на нужную ветку производится командой

```
git checkout имя_новой_ветки
```
Проверить актуальное местоположение можно командой
```
git branch
```
Звездой (*) обозначена та ветка, на которой находимся на данный момент.

## 12. Слияние ветки с главной
После внесения изменений в файл и их сохранения в рабочей ветке, необходимо слияние ветки с главной веткой. Это делается, используя команду
```
git merge <имя ветки>
```

## 13. Удаление веток
Для удаления ненужной ветки, используем
```
git branch -d <имя ветки>
```
## 14. Работа с изображениями
Чтобы вставить изображение в текст,
1. необходимо в самом тексте написать следующее
```
![текст который выведется на экран, если изображение по каким то причинам не добавится](имя файла изображения)
```
2. создаем в отслеживаемой папке файл 
```
.gitignore
```
3. добавляем в .gitignore имя изображения, которое необходимо "игнорировать"
также можно вставить расширение изображения, которое нужно "игнорировать"



# Работа с удаленными репозиториями

Чтобы связать локальный репозиторий с удаленным, нужно в первую очередь создать проект на github.com. Для этого

1. создаем аккаунт
2. во вкладке *repositories* жмем *new*
3. заполняем пункты *repository name* и остальные опции
4. жмем *create repositoty*

Далее открывается страница с пошаговыми командами git-а для создания локального репозитория или, если он уже создан, связывания локального репозитория с удаленным.

1. создание локального репозитория,связанного с удаленным
```
git init
git add README.md
git commit -m "first commit"
git branch -M main
git remote add origin https://ссылка_на_ваш_проект на github
git push -u origin main
```
2. создание связи с уже созданным локальным репозиторием
```
git remote add origin https://ссылка_на_ваш_проект на github
git branch -M main
git push -u origin main
```
3. или советы для импорта кода из других репозиториев

Переходим на *github* обновляем и проверяем - проект должен появиться на удаленном репозитории.

Когда вносятся изменения в проект на удаленном репозитории после сохранения изменений, для получения их на локальный репозиторий в git выполняется команда 
```
git pull
```
Когда вносим изменения в проект на локальном репозитории, чтобы отправить эти изменения на удаленный репозиторий после очередного commit-а выполняется команда
```
git push
```
Когда возникает конфликт, при слиянии удаленного и локального репозитория, разрешаем его, следуя подсказкам git-а.


## Работа c публичным репозиторием

В проектах с открытыми исходными кодами другие люди тоже могут поучаствовать в проекте, то есть вы можете предложить свои идеи.  

В git  это делается с помощью функции ***pull request***
Для этого выполняется сделующая последовательность действий.

1. На сайте github нелаем форк (*fork*) интересующего нас репозитория.

2. Выполняется команда
```
git clone
``` 
для нашей версии этого репозитория

3. Создаем ветку с предлагаемыми изменениями

```
git branch имя_ветки
```

4. Производим все изменения только в этой ветке
5. Отправляем эти изменения на свой аккаунт командой 

```
push
```

6. На Github появляется возможность отправить pull request.
Если pull request не появился, то ищем ее в 
```
pull requests - new pull requests - в head repository выбираем новую ветку - create pull request
```
